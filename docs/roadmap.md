# AIEngineerBlueprint Roadmap

The static platform is in place. The next work is content depth, not more scaffolding.

## Phase 1: Deepen priority tracks

Focus first on:

- LLM Fundamentals
- RAG
- MCP
- AI Agents
- Agentic Systems
- Guardrails
- AI Security
- Production AI Systems
- Building Products

Each chapter should use plain-English explanations, analogies, topic-specific flow diagrams, realistic failure cases, hands-on labs, production checklists, and references.

## Phase 2: Build complete product labs

Add end-to-end builds for:

- Enterprise RAG
- AI code reviewer
- Document intelligence
- AI support bot
- Engineering assistant

Each product should include requirements, architecture, API design, data model, prompt library, evals, guardrails, CI/CD, deployment, monitoring, cost controls, and security review notes.

## Phase 3: Add richer interactivity

Create browser-only simulations for:

- Tokenization
- Embedding similarity
- RAG chunking and retrieval
- Agent planning loops
- Guardrail policy checks
- Eval scoring
- Cost and latency estimation

## Phase 4: Publish and iterate

Push to `AIEngineerBlueprint.github.io`, enable GitHub Pages, test offline mode on the live domain, then improve content in small repeatable passes.
