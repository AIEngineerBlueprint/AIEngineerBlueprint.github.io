Learning track

OpenAI Codex

A production-focused path for mastering OpenAI Codex through concepts, diagrams, hands-on labs, interview practice, and capstone work.

Track outcomes

By the end of this track, you should be able to reason from first principles, design the architecture, identify failure modes, build a working slice, and explain how the system behaves in production.

In plain words: Codex-style agents are like task-focused coding workers: give them a repository, a goal, constraints, and a validation path. They are most useful when the work can be expressed as a verifiable code change rather than an open-ended conversation.

Chapter 1 Codex overview OpenAI Codex is an agentic coding tool available as a CLI, IDE extension, and cloud/web interface that reads a codebase, plans changes, edits files, and runs commands, with configurable levels of autonomy through sandboxing and approval policies. Chapter 2 Model selection Model selection chooses the smallest, safest, fastest model that satisfies the product's quality and operational requirements. Chapter 3 Prompting Prompting Codex effectively means giving it a specific, scoped task description plus the repository context it needs, via AGENTS.md or explicit file references, since a vague instruction produces a vague or overly broad plan. Chapter 4 Agent workflows Agent workflows for Codex are patterns for structuring how a task moves from description to a mergeable change: local CLI sessions for interactive work, and cloud or web sessions for longer-running or delegated tasks that continue running after you step away. Chapter 5 Repository setup Repository setup for Codex means adding an AGENTS.md file describing build commands, conventions, and constraints, following a hierarchical discovery order from global to project-specific, so Codex starts every session with consistent expectations. Chapter 6 Testing Testing with Codex means having it run the project's real test suite after making changes and iterate based on actual failures, closing the loop between generating a change and verifying it rather than trusting the diff alone. Chapter 7 Refactoring Refactoring with Codex means using it to restructure code without changing behavior, verified by running the existing test suite before and after, with the task scoped to an explicit set of files to keep the diff reviewable. Chapter 8 Review Codex's code review feature reads a pull request's diff on GitHub and generates review comments flagging likely bugs, missing edge cases, and convention violations, intended to give human reviewers a useful first pass rather than replace their approval. Chapter 9 Security Security for Codex centers on sandbox mode, read-only, workspace-write, or danger-full-access, and approval policy, untrusted, on-request, or never, which together define exactly what the agent can touch and when it needs explicit human confirmation. Chapter 10 Automation Automation with Codex means running it non-interactively as part of CI or a scheduled job, typically through cloud sessions, to perform a scoped task like dependency updates or issue triage without a human present for each step. Chapter 11 Team workflow Team workflow for Codex means establishing shared conventions, a checked-in AGENTS.md, and consistent review standards so every developer's use of the tool produces predictable, reviewable results rather than each person developing incompatible habits. Chapter 12 Best practices Best practices for Codex combine scoped prompting, an up-to-date AGENTS.md, sandbox and approval settings matched to task risk, and mandatory test verification into a consistent habit, so output quality doesn't depend on any one developer's individual discipline. Chapter 13 Troubleshooting Troubleshooting Codex means diagnosing whether an unexpected result comes from insufficient repository context, a sandbox or approval setting blocking an action, or a genuinely wrong plan, and applying the fix that matches the actual cause rather than just rewording the prompt. Chapter 14 Codex lab The Codex lab is a hands-on exercise to run one real, moderately complex task through Codex end to end, from a scoped prompt through test verification, then reflect on what sandbox, approval, or AGENTS.md configuration would improve the next run. Chapter 15 Mini project The mini project ships one small, complete Codex workflow artifact for a real repository, such as an AGENTS.md plus a documented review checklist, that a teammate could adopt directly. Chapter 16 Capstone The capstone designs a complete operating model for using Codex across a real engineering team: which tasks run autonomously versus require review, what sandbox and approval settings apply by default, shared AGENTS.md standards, and the metrics that prove it's actually helping. Chapter 17 Interview pack The Codex interview pack prepares you to discuss agentic coding tool adoption credibly: sandbox and approval configuration, AGENTS.md design, and rollout risk management, backed by a concrete story rather than a feature list.
