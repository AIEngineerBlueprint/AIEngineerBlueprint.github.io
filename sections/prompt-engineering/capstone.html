Prompt Engineering · Chapter 16

Capstone

The capstone designs a complete prompt engineering system for a real product: a prompt library with versioning, a structured prompting standard, a testing process, and an evaluation-gated promotion process, as one coherent system.

Calculating reading timeBuild project

What you're building, and why

In practice it comes down to combining a documented prompt library with versioned templates, a structured prompting standard applied consistently, a labeled testing process for every prompt change, and an evaluation-gated process for promoting a new prompt version to production into one complete system.

It is like designing a publication's complete editorial process, not just writing one good article; the process is what keeps quality consistent across every piece produced.

The brief

Write a one-page prompt engineering system design for a real product: prompt library structure, versioning process, testing standard, and evaluation gate.

A worked reference

A capstone for a customer-facing assistant might specify a versioned prompt library, a structured output schema for every prompt, a 30-case test set per major prompt, and a rule that no prompt version reaches production without passing its evaluation set.

Starting point in code

const promptSystem = {
  library: "versioned templates, one owner each",
  structure: "labeled sections + JSON output schema",
  testing: "30-case set per major prompt",
  promotionRule: "no version ships without passing its eval set"
};

Change the inputs and watch which assumption breaks first. The gap between what you expected and what you got is where the learning is.

Trade-offs to document

  • The core tension: A complete prompt engineering system takes real process design effort beyond writing individual good prompts, but it prevents the common failure of prompt quality becoming inconsistent and unmanageable as a product and team grow

Definition of done

Watch out for

  • A prompt engineering practice is not complete once a few prompts work well in testing; without a shared library, versioning, and an evaluation-gated promotion process, prompt quality becomes inconsistent and hard to maintain as more people and prompts are involved.
