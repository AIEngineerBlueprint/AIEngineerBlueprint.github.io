Learning track

LLM Fundamentals

A production-focused path for mastering LLM Fundamentals through concepts, diagrams, hands-on labs, interview practice, and capstone work.

Track outcomes

By the end of this track, you should be able to reason from first principles, design the architecture, identify failure modes, build a working slice, and explain how the system behaves in production.

In plain words: Think of an LLM as a very fast technical editor. It reads the words already on the page, predicts what should come next, and keeps revising that prediction as more context appears. This matters because every AI product eventually pays for tokens, latency, context quality, and model behavior. If you understand the inference path, you can debug bad answers instead of guessing.

The running example

Throughout this track, imagine you own the model gateway for a product team: every complaint about cost, latency, weird answers, or flaky behavior lands on your desk. Each chapter is one of the dials you'll be asked to explain or turn.

Production build path

Use this track to contribute to one of the platform projects below. These projects are intentionally track-level because they combine multiple chapters into a complete product slice.

Chapter 1 LLM mental model A large language model is a neural network trained to predict the next token given all previous tokens. Chapter 2 Pretraining Pretraining is the first and largest phase of LLM training, where the model learns to predict the next token across hundreds of billions of tokens of text from the internet, books, and code — with no human-provided labels. Chapter 3 Instruction tuning Instruction tuning is the training phase that turns a raw text-completion model into one that answers questions and follows directions — by fine-tuning on curated instruction-response pairs until 'continue the text' becomes 'do what was asked'. Chapter 4 RLHF Reinforcement Learning from Human Feedback (RLHF) is a technique that fine-tunes a language model using human preferences. Chapter 5 Fine tuning Fine-tuning takes a pretrained model and continues training it on a smaller, task-specific dataset, adjusting weights to make the model better at a particular behaviour or domain. Chapter 6 Fine-tuning vs training Fine-tuning adapts an existing model to a narrower behavior or task, while training or pretraining builds broad model capability from large-scale data. Chapter 7 LoRA LoRA (Low-Rank Adaptation) is a parameter-efficient fine-tuning method that adds small trainable rank-decomposition matrices to each transformer layer instead of updating all model weights. Chapter 8 MoE Mixture of Experts is an architecture where each token activates only a few specialized sub-networks (experts) out of many, letting a model carry enormous total parameters while spending only a fraction of them per token. Chapter 9 Context windows The context window is the maximum number of tokens a model can process in a single forward pass — both prompt and response combined. Chapter 10 Inference pipeline The inference pipeline is the sequence every request travels — tokenize, prefill the prompt, decode token-by-token, detokenize and stream — and naming its stages is what turns vague latency complaints into locatable problems. Chapter 11 GPU memory GPU memory budgeting is the capacity math of LLM serving: weights, KV cache, and activations compete for fixed VRAM, and the KV cache — growing with every concurrent user and every token of context — is the term that ends up deciding how many users fit on a card. Chapter 12 Model loading Model loading is the sequence that takes weights from artifact to accepting traffic — fetch, verify, place onto GPUs, warm up — and getting it wrong shows up as ninety-second 'ready' lies, silent version drift, and autoscaling that arrives after the traffic spike leaves. Chapter 13 Distributed inference Distributed inference serves models too large or too popular for one GPU by splitting the work — tensor parallelism within layers, pipeline parallelism across layers, and replication across the fleet — where the topology chosen decides latency, throughput, and what breaks when a node dies. Chapter 14 Context compression Context compression shrinks what goes into the window — summarizing history, pruning and deduplicating retrieved chunks, trimming boilerplate — to cut cost and latency while keeping the evidence the answer actually needs; done carelessly, it deletes exactly that evidence. Chapter 15 Prompt caching Prompt caching lets providers reuse the computed state of a repeated prompt prefix — the system prompt, tool schemas, and shared examples — so you stop paying full prefill cost and latency to reprocess the same tokens on every request. Chapter 16 Sampling Sampling is the process of choosing the next token during generation. Chapter 17 Temperature Temperature is a scalar applied to the model's output probabilities before sampling. Chapter 18 Top-K and Top-P Top-K and Top-P are the sampling filters that decide which candidate tokens survive before the random draw — Top-K keeps a fixed number of the most likely tokens, Top-P keeps the smallest set whose probabilities sum past a threshold — and together with temperature they define your model's randomness contract. Chapter 19 Reasoning models Reasoning models (like o1, o3, DeepSeek-R1) are LLMs trained with extended chain-of-thought that produces a hidden or visible 'thinking' trace before generating a final answer. Chapter 20 LLM lab The LLM lab is a hands-on exercise to send the same prompt to a model at different temperature and sampling settings, and observe concretely how the output's consistency and creativity change. Chapter 21 Mini project The mini project ships a small, complete LLM-powered feature for one real task, with a documented model and sampling choice, and an explanation of one inference-level trade-off, like context length or caching, that shaped the design. Chapter 22 Capstone The capstone designs a complete LLM-powered system for a real use case: model selection justified by task fit, an inference architecture accounting for context and caching, sampling strategy, and a plan for the trade-offs between latency, cost, and quality. Chapter 23 Interview pack The LLM fundamentals interview pack prepares you to explain pretraining, fine-tuning, sampling, and inference mechanics with concrete, mechanistic reasoning, backed by the ability to reason through a described model-selection or performance scenario.
