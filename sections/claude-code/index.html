Learning track

Claude Code

A production-focused path for mastering Claude Code through concepts, diagrams, hands-on labs, interview practice, and capstone work.

Track outcomes

By the end of this track, you should be able to reason from first principles, design the architecture, identify failure modes, build a working slice, and explain how the system behaves in production.

In plain words: Claude Code is like a pair programmer that can read the repo, run tools, make edits, and remember project conventions when you give it a clear operating lane. The value comes from workflow design: small tasks can be direct, larger tasks need plans, review, permissions, and verification.

Chapter 1 Claude Code overview Claude Code is Anthropic's agentic coding CLI that reads a repository, plans a change, edits files, runs commands, and iterates inside the developer's own terminal, IDE, or CI environment, instead of only answering questions in a chat window. Chapter 2 Command reference Claude Code's command surface includes slash commands typed inside a session, such as /clear, /compact, /model, and /agents, plus CLI flags used to launch or script it, such as -p for headless print mode. Chapter 3 Project setup Project setup for Claude Code means writing a CLAUDE.md memory file and a .claude/settings.json configuration so the agent starts every session already knowing how to build, test, and navigate the repository, instead of rediscovering it each time. Chapter 4 Agent workflows Agent workflows in Claude Code are patterns for structuring larger tasks: using the main session for direct work, spawning subagents for parallelizable or context-heavy research, and using plan mode to align on an approach before any file is touched. Chapter 5 Context management Context management is the discipline of controlling what enters Claude Code's limited context window so the model reasons from relevant, current information instead of stale or irrelevant history. Chapter 6 Testing workflows Testing workflows use Claude Code's ability to actually execute a project's test suite, read the failure output, and revise its own edit, closing the loop between writing code and verifying it instead of leaving verification to the human. Chapter 7 Refactoring Refactoring with Claude Code means using it to restructure code without changing behavior, verified by running the existing test suite before and after the change, not just by reading the diff. Chapter 8 Code review Code review with Claude Code means using the agent to read a diff and surface correctness bugs, missing edge cases, and security concerns before a human reviewer spends time on it, not to replace human sign-off. Chapter 9 Security practices Security practices for Claude Code configure what the agent is allowed to touch: which directories, which shell commands, and which secrets it can access, so agentic capability does not become an uncontrolled attack surface. Chapter 10 Enterprise usage Enterprise usage of Claude Code involves organization-wide managed settings, centralized permission policy, and audit logging so security and platform teams can enforce consistent rules across every developer's agent, rather than relying on each developer configuring safety individually. Chapter 11 Team standards Team standards for Claude Code means committing a shared CLAUDE.md, custom slash commands, and settings to source control so agentic workflows are consistent, reviewable, and improve over time as a team artifact, not tribal knowledge held by one developer. Chapter 12 Automation Automation with Claude Code means running it non-interactively, typically in CI or a script, to perform a scoped task like generating a summary, checking a diff against a policy, or drafting release notes, without a human present to approve each step in real time. Chapter 13 Troubleshooting Troubleshooting Claude Code sessions means diagnosing why an agent produced a bad result: was it missing context, blocked by a permission it needed, confused by stale conversation history, or simply wrong, and applying the fix that matches the actual cause. Chapter 14 Claude lab The Claude Code lab is a hands-on exercise where you run one real, moderately complex task end to end through the agent, from planning to tests passing, then critique the workflow itself rather than just the code output. Chapter 15 Mini project The mini project asks you to ship one complete, reusable Claude Code workflow artifact, such as a project CLAUDE.md, a custom command, or a CI automation, that a teammate could adopt immediately. Chapter 16 Capstone The capstone project designs a full operating model for using Claude Code across a real engineering team: scope of allowed autonomous work, required review gates, shared configuration, and the metrics that prove it is actually improving delivery rather than just increasing AI usage. Chapter 17 Interview pack The Claude Code interview pack prepares you to discuss agentic coding tools credibly in interviews: what you configured, what went wrong, how you scoped risk, and what technical trade-offs you understand deeply enough to defend under follow-up questions.
