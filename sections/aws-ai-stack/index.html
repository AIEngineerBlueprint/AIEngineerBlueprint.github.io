Learning track

AWS AI Stack

A production-focused path for mastering AWS AI Stack through concepts, diagrams, hands-on labs, interview practice, and capstone work.

Track outcomes

By the end of this track, you should be able to reason from first principles, design the architecture, identify failure modes, build a working slice, and explain how the system behaves in production.

In plain words: The AWS AI stack is like a warehouse of managed building blocks: model access, storage, search, compute, identity, logs, and deployment lanes. Knowing the services helps you design secure and operable AI products without building every platform component yourself.

Chapter 1 AWS overview AWS AI architecture combines managed model services, compute, storage, search, identity, networking, and observability into production AI workflows. Chapter 2 Bedrock Amazon Bedrock is a managed AWS service for accessing foundation models, building generative AI applications, and integrating enterprise controls. Chapter 3 SageMaker SageMaker is AWS's managed platform for building, training, deploying, and monitoring machine-learning models. Chapter 4 Lambda AWS Lambda runs short-lived serverless functions that can glue AI workflows, APIs, events, and background processing. Chapter 5 ECS Amazon ECS is a managed container orchestration service that runs Docker containers as tasks and services across EC2 or Fargate compute, commonly used to host model-serving APIs, RAG backends, and agent orchestration layers. Chapter 6 EKS Amazon EKS is a managed Kubernetes control plane for running containerized workloads, commonly used for AI systems that need GPU node pools, complex scheduling, or portability across cloud and on-prem Kubernetes clusters. Chapter 7 OpenSearch Amazon OpenSearch Service is a managed search and analytics engine that supports both traditional text search and vector similarity search through its k-NN plugin, making it usable as a retrieval backend for RAG or as a log and metrics analytics store. Chapter 8 Aurora Amazon Aurora is a managed relational database compatible with MySQL and PostgreSQL, commonly used in AI systems for transactional application data and, via the pgvector extension on Aurora PostgreSQL, as a lightweight vector store colocated with other relational data. Chapter 9 S3 Amazon S3 is object storage used across AI systems to hold training datasets, model artifacts, document sources for RAG ingestion, and logs, with lifecycle rules and access controls that scale from a few files to petabytes. Chapter 10 CloudWatch Amazon CloudWatch collects metrics, logs, and alarms across AWS services, giving AI systems visibility into latency, error rate, cost-relevant usage, and model-serving health in one place. Chapter 11 IAM AWS IAM controls which users, services, and roles can perform which actions on which resources, forming the permission boundary around every other AWS service an AI system touches, from Bedrock model access to S3 document buckets. Chapter 12 Reference deployment A reference deployment for the AWS AI stack is a concrete, opinionated architecture combining compute, storage, search, and identity services into one coherent, deployable system rather than a list of services considered in isolation. Chapter 13 Cost controls Cost controls for the AWS AI stack are the deliberate practices and tools, such as budgets, tagging, and right-sizing, that keep spend on compute, model inference, and storage predictable and attributable, since AI workloads can scale cost quickly without corresponding scaling in value. Chapter 14 AWS lab The AWS lab is a hands-on exercise to deploy one small AI workload across at least three real AWS services, such as Lambda, S3, and Bedrock, configure IAM roles between them, and verify it end to end with a real request. Chapter 15 Mini project The mini project ships one small, real AI feature deployed on AWS with documented IAM roles, cost tags, and monitoring, that another engineer could redeploy from the instructions alone. Chapter 16 Capstone The capstone designs a complete AWS AI reference architecture for a real product: compute choice, storage and retrieval layer, IAM boundary design, observability plan, and a cost-control strategy, as one coherent deployable system. Chapter 17 Interview pack The AWS AI stack interview pack prepares you to discuss real AWS deployment decisions credibly: why you chose specific services, how you scoped IAM permissions, and how you controlled cost, backed by a concrete story rather than a list of service names.
