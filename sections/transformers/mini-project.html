Transformers · Chapter 15

Mini project

The mini project implements or clearly explains one small transformer component, such as a single attention head or a positional encoding scheme, from scratch, with a working example that demonstrates it operating on real input.

Calculating reading timeBuild project

What you're building, and why

In practice it comes down to implementing or precisely tracing through one transformer component in isolation, such as scaled dot-product attention for a small sequence, and demonstrating it produces the expected output on a concrete example.

It is like building a working scale model of one specific part of a larger machine, to prove you understand exactly how that part functions.

The brief

Implement scaled dot-product attention from scratch for a small example sequence, and verify your computed attention weights match a hand-calculated expectation.

A worked reference

A mini project might implement scaled dot-product attention from scratch in a few dozen lines of code, run it on a 5-token example sequence, and verify the attention weights match hand-calculated expectations.

Starting point in code

function scaledDotProductAttention(Q, K, V) {
  const scores = matmul(Q, transpose(K)) / Math.sqrt(K[0].length);
  const weights = softmax(scores);
  return matmul(weights, V);
}

Change the inputs and watch which assumption breaks first. The gap between what you expected and what you got is where the learning is.

Trade-offs to document

  • The core tension: Implementing one component thoroughly from scratch builds deeper understanding than describing the whole transformer architecture at a high level, though it takes more focused time than a broader but shallower overview

Definition of done

Watch out for

  • Being able to name transformer components like attention and feed-forward networks is not the same as being able to implement or precisely trace through one of them; the mini project's value is in that concrete, verifiable understanding.
