Learning track

Local AI

A production-focused path for mastering Local AI through concepts, diagrams, hands-on labs, interview practice, and capstone work.

Track outcomes

By the end of this track, you should be able to reason from first principles, design the architecture, identify failure modes, build a working slice, and explain how the system behaves in production.

In plain words: Local AI is like running a small workshop on your own laptop instead of renting a large factory in the cloud. You gain privacy and control, but capacity is limited. Local models are useful for offline development, privacy-sensitive workflows, demos, and cost control.

Chapter 1 Local AI overview Local AI runs models on developer machines or private infrastructure instead of relying entirely on hosted APIs. Chapter 2 Ollama Ollama runs local language models behind a simple developer API, making offline experimentation and private local workflows easier. Chapter 3 Kimi Kimi is a family of large language models, including open-weight releases, that can be run locally or self-hosted, offering an alternative to closed, API-only models for teams that need to run inference on their own infrastructure. Chapter 4 LM Studio LM Studio is a desktop application that lets developers download, run, and experiment with open-weight language models locally through a graphical interface, without needing to manually configure a runtime from the command line. Chapter 5 GGUF GGUF is a file format commonly used to package quantized local LLM weights and metadata for efficient inference engines. Chapter 6 Quantization Quantization stores model weights or activations with fewer bits to reduce memory and speed inference, usually with some quality trade-off. Chapter 7 CPU vs GPU CPU vs GPU selection determines whether local or production inference optimizes for availability, cost, throughput, or latency. Chapter 8 Model selection Model selection chooses the smallest, safest, fastest model that satisfies the product's quality and operational requirements. Chapter 9 Offline development Offline development means building and testing an AI feature using a local model with no dependency on an internet connection or a cloud API, useful for privacy-sensitive work, unreliable connectivity, or reducing development-time API costs. Chapter 10 Privacy Privacy as a driver for local AI means keeping sensitive data on infrastructure the organization controls, rather than sending it to a third-party API, which matters for data that's regulated, confidential, or contractually restricted from leaving a specific environment. Chapter 11 Performance Performance for local AI covers the practical latency and throughput a model achieves on real local hardware, which varies enormously by model size, quantization level, and whether inference runs on CPU or GPU. Chapter 12 Developer workflow Developer workflow for local AI covers how local models fit into a team's daily development process, such as using a local model for fast iteration and a cloud model for final validation, or switching between them through a consistent interface. Chapter 13 Packaging Packaging for local AI means bundling a model, its runtime, and its dependencies into a distributable, reproducible unit so it can be deployed consistently across different machines without each one requiring manual setup. Chapter 14 Local AI lab The local AI lab is a hands-on exercise to run one real model locally, measure its actual tokens-per-second performance on your hardware, and compare quality and speed across two different quantization levels. Chapter 15 Mini project The mini project ships a small, complete local AI feature, packaged and documented, with a measured performance profile and a clear privacy justification, that another engineer could deploy and run. Chapter 16 Capstone The capstone designs a complete local AI deployment for a real use case: model and hardware choice justified by measured performance, a packaging and reproducibility plan, a privacy justification, and a developer workflow for iterating against it. Chapter 17 Interview pack The local AI interview pack prepares you to discuss local model deployment credibly: hardware and quantization trade-offs, privacy-driven decisions, and packaging for reproducibility, backed by a concrete story rather than general enthusiasm for running models locally.
