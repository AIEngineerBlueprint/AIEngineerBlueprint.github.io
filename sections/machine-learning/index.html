Learning track

Machine Learning

A production-focused path for mastering Machine Learning through concepts, diagrams, hands-on labs, interview practice, and capstone work.

Track outcomes

By the end of this track, you should be able to reason from first principles, design the architecture, identify failure modes, build a working slice, and explain how the system behaves in production.

In plain words: Machine learning is like training a new teammate with labeled examples. Instead of writing every rule, you show enough examples until the teammate learns the pattern. ML is the foundation for understanding why models need data, metrics, validation, and monitoring.

Chapter 1 What is ML? Machine learning is a branch of AI where a system learns to make predictions or decisions by finding patterns in historical data, rather than following explicitly programmed rules for every case. Chapter 2 Supervised learning Supervised learning trains a model using labelled examples — pairs of input and correct output — so the model learns to predict the output for new inputs. Chapter 3 Unsupervised learning Unsupervised learning finds structure or patterns in data without labeled outputs, such as grouping similar items together or reducing data to its most important dimensions, distinct from supervised learning which learns from labeled input-output pairs. Chapter 4 Feature engineering Feature engineering is the process of selecting, transforming, or creating the specific input signals a model actually learns from, since raw data rarely arrives in the form that best exposes the patterns a model needs to find. Chapter 5 Training data Training data is the labeled examples a supervised model learns from, and its quality, representativeness, and labeling accuracy directly determine the ceiling on how good the resulting model can possibly be. Chapter 6 Loss functions A loss function is a mathematical measure of how wrong a model's prediction is. Chapter 7 Optimization Optimization in machine learning is the process of adjusting a model's parameters to minimize a loss function, using an algorithm like gradient descent that iteratively moves parameters in the direction that reduces prediction error. Chapter 8 Validation Validation checks whether AI inputs, outputs, or tool calls satisfy explicit rules before the system trusts or acts on them. Chapter 9 Overfitting Overfitting happens when a model learns the training data too precisely — including its noise and quirks — and performs worse on new, unseen data. Chapter 10 Model selection Model selection chooses the smallest, safest, fastest model that satisfies the product's quality and operational requirements. Chapter 11 Metrics Metrics are the specific, quantitative measures used to evaluate a model's performance, chosen based on what actually matters for the task, since a single generic accuracy number can be misleading or even meaningless for some problems. Chapter 12 Experiment tracking Experiment tracking means systematically recording each model training run's configuration, data version, and resulting metrics, so different approaches can be compared reliably and past results can be reproduced. Chapter 13 Deployment patterns Deployment patterns are the different ways a trained machine learning model is actually put into production use, such as batch predictions run periodically, real-time predictions served on demand, or embedded predictions running directly on a user's device. Chapter 14 ML lab The ML lab is a hands-on exercise to train a simple model on a real small dataset, evaluate it with an appropriate metric, then engineer one new feature and measure whether it actually improves the result. Chapter 15 Mini project The mini project ships a small, complete ML model for one real task, with documented feature engineering, an appropriate evaluation metric, and tracked experiments, that another engineer could reproduce. Chapter 16 Capstone The capstone designs a complete ML workflow for a real problem: training data assessment, feature engineering, optimization and metric choices, experiment tracking, and a deployment pattern, as one coherent system. Chapter 17 Interview pack The machine learning interview pack prepares you to discuss metric selection, feature engineering, and training data quality with concrete reasoning, backed by the ability to diagnose a described model performance problem.
