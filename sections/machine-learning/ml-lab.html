Machine Learning · Chapter 14

ML lab

The ML lab is a hands-on exercise to train a simple model on a real small dataset, evaluate it with an appropriate metric, then engineer one new feature and measure whether it actually improves the result.

Calculating reading timeBuild project

What you're building, and why

In practice it comes down to training a baseline model on a small real dataset, evaluating it with a metric matched to the task, then adding one engineered feature and re-evaluating to see whether it measurably improved performance.

It is like running a controlled before-and-after experiment on your own recipe, changing one ingredient at a time to see its actual effect.

The brief

Train a baseline model on one real dataset, measure it with an appropriate metric, then add one engineered feature and measure whether it actually improved the result.

A worked reference

A lab might train a baseline model to predict customer churn, measure precision and recall, then add a feature like days since last purchase and re-measure to see if it improved recall on the churn class specifically.

Starting point in code

const baseline = trainAndEval(features_v1);
const withNewFeature = trainAndEval([...features_v1, "days_since_last_purchase"]);
console.log({ baseline, withNewFeature }); // did recall actually improve?

Change the inputs and watch which assumption breaks first. The gap between what you expected and what you got is where the learning is.

Trade-offs to document

  • The core tension: Testing one feature at a time with a real before-and-after comparison takes more discipline than adding several changes at once, but it's the only way to know which specific change actually helped

Definition of done

Watch out for

  • The lab is not complete once a model trains and produces some accuracy number; without comparing before and after an engineered feature using an appropriate metric, you have no evidence the feature actually helped.
