MCP · Chapter 14

MCP lab

The MCP lab is a hands-on exercise to build a small MCP server exposing one resource and one tool, connect it to a client, and verify both work correctly end to end.

Calculating reading timeBuild project

What you're building, and why

In practice it comes down to implementing a minimal MCP server with one resource endpoint and one tool with a schema, connecting a client to it, and testing that the client can both read the resource and successfully invoke the tool.

It is like wiring a small working circuit with one sensor and one actuator, to prove you understand the full connection before building something larger.

The brief

Build a minimal MCP server with one resource and one tool, connect a client to it, and verify both work correctly.

A worked reference

A lab might build an MCP server exposing a list_files resource and a read_file tool, connect a client, and verify it can both list and read a real file through the protocol.

Starting point in code

server.resource("files://{path}", async (path) => readFile(path));
server.tool("read_file", { parameters: { path: "string" }, handler: readFile });
// Connect a client, verify it can both list/read resources AND call the tool

Change the inputs and watch which assumption breaks first. The gap between what you expected and what you got is where the learning is.

Production build path

Use this chapter as a building block in one of the platform projects below. The learning target is not memorizing the term; it is proving you can design, implement, evaluate, and operate it inside a product.

Trade-offs to document

  • The core tension: Building a real minimal server and connecting a real client takes more setup than reading about the protocol's message format, but it's the only way to verify you understand how the pieces actually connect

Definition of done

Watch out for

  • The lab is not complete once the server code compiles; without actually connecting a client and verifying both the resource and the tool work end to end, you have no evidence the implementation is correct.
