Deep Learning · Chapter 14

DL lab

The deep learning lab is a hands-on exercise to train a small network with one regularization technique, deliberately compare it against an unregularized version, and observe the difference in the train-validation gap.

Calculating reading timeBuild project

What you're building, and why

In practice it comes down to training the same small network twice, once without regularization and once with a technique like dropout, and comparing training loss versus validation loss for both to see the overfitting gap directly.

It is like running a controlled experiment with and without one variable changed, to see its actual effect rather than assuming it works.

The brief

Train a small network with and without one regularization technique, and compare the gap between training and validation performance in each case.

A worked reference

A lab might train a small image classifier on a limited dataset twice, once with no dropout and once with 20 percent dropout, and compare the gap between training and validation accuracy in each case.

Starting point in code

const withoutDropout = trainModel({ dropout: 0 });
const withDropout = trainModel({ dropout: 0.2 });
console.log({
  withoutDropout: { train: withoutDropout.trainAcc, val: withoutDropout.valAcc },
  withDropout: { train: withDropout.trainAcc, val: withDropout.valAcc }
});

Change the inputs and watch which assumption breaks first. The gap between what you expected and what you got is where the learning is.

Trade-offs to document

  • The core tension: Running the same training twice to compare with and without a technique takes more compute time than training once, but it's the only way to see the actual effect rather than assuming a technique like dropout is helping

Definition of done

Watch out for

  • The lab is not complete once a model trains successfully with regularization added; the point is comparing it against the unregularized version to see whether it actually reduced the train-validation gap.
