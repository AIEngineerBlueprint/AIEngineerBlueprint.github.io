Agentic AI · Chapter 15

Agentic lab

The agentic lab is a hands-on exercise to build a small but complete agentic workflow: task decomposition, at least one human checkpoint, and a retry path, then verify it against a deliberately injected failure.

Calculating reading timeBuild project

What you're building, and why

In practice it comes down to building a small multi-step agentic workflow end to end with at least one human checkpoint and one injected failure, then observing whether the retry, checkpoint, and recovery logic actually behave as designed.

It is like a controlled fire drill for a small system you built yourself, so you can see the safety mechanisms actually engage.

The brief

Build a three-step agentic workflow with one human checkpoint, inject a failure at the middle step, and verify the workflow escalates rather than silently continuing.

A worked reference

A lab implementation might build a three-step file-processing agent with a checkpoint before a destructive action, then inject a failure at step two to confirm the retry and escalation logic actually triggers.

Starting point in code

// Lab: 3-step workflow with one checkpoint and one injected failure.
const steps = ["fetch", "transform", "write"];
// inject a failure at "transform" and confirm escalation, not silent continue
const result = await runWorkflow(steps, { injectFailureAt: "transform" });
assert(result.status === "escalated_to_human");

Change the inputs and watch which assumption breaks first. The gap between what you expected and what you got is where the learning is.

Trade-offs to document

  • The core tension: Building the full workflow including failure injection takes longer than a happy-path demo, but it is the only way to verify the checkpoint and retry logic actually works instead of just looking correct on paper

Definition of done

Watch out for

  • The lab is not complete when the happy path runs successfully once; the checkpoint and retry logic must be proven against an actual injected failure, not just described.
