Neuron model

A neuron in a neural network is a small computational unit that takes a weighted sum of its inputs, adds a bias term, and passes the result through a nonlinear activation function, producing one output value that feeds into the next layer. This chapter turns that idea into a traceable learning slice with examples, checks, and failure modes.

Beginner explanation

If you have no AI background, start here: A neuron in a neural network is a small computational unit that takes a weighted sum of its inputs, adds a bias term, and passes the result through a nonlinear activation function, producing one output value that feeds into the next layer.

Analogy: It is like a single voter who weighs several pieces of evidence with different importance, adds their own baseline lean, and then decides yes or no rather than reporting a raw undecided score.

Trade-offs

  • Quality vs latency: The main trade-off is a single neuron can only represent a limited, essentially linear-plus-threshold decision boundary; real predictive power comes from combining many neurons across layers, not from making any one neuron more complex
  • Flexibility vs reliability: Open-ended designs adapt to more inputs; constrained contracts are easier to test and operate.
  • Automation vs oversight: More automation reduces manual work but needs stronger logging, review points, and rollback paths.

Small runnable example

Use this snippet as the smallest concrete anchor for the concept. Change the inputs, then observe which assumptions fail.

function neuron(inputs, weights, bias, activation) {
  const z = inputs.reduce((sum, x, i) => sum + x * weights[i], bias);
  return activation(z); // e.g. sigmoid, relu
}
