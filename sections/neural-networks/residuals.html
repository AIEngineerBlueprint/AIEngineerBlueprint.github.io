Neural Networks · Chapter 7

Residuals

Residual connections add a layer's input directly to its output, letting the network learn only the difference, or residual, between input and desired output, which makes very deep networks dramatically easier to train.

Calculating reading timeHands-on labQuiz

In plain terms

It is like editing a document by marking only the changes you want to make to the existing text, rather than rewriting the entire document from scratch for every revision.

How it works

In practice it comes down to adding the identity of a layer's input to the output of that layer's transformation, so the layer only needs to learn a residual adjustment rather than the full mapping from scratch, which lets gradients flow through the identity path directly to earlier layers during backpropagation.

A concrete example

A very deep image classification network uses residual connections so that if a particular set of layers isn't helpful for a given input, the network can effectively let the identity path dominate, passing information through mostly unchanged, rather than being forced to learn a full non-trivial transformation at every single layer.

Trade-offs

  • The core tension: Residual connections make very deep networks trainable where they would otherwise fail due to vanishing gradients, but they add a small amount of extra computation and require the input and output dimensions of a residual block to match or be explicitly reshaped

Try it in code

function residualBlock(x, transform) {
  return add(x, transform(x)); // learn only the residual, not the full mapping
}

Change the inputs and watch which assumption breaks first. The gap between what you expected and what you got is where the learning is.

Hands-on lab

Task: Explain in your own words why a network with residual connections can be trained successfully at 100+ layers deep, while an equivalent network without them often cannot.

  1. Build or configure it, using the code example above as your starting point.
  2. Test it against one normal case and one edge case, and write down both actual outputs.
  3. Note what you would log, and which single metric would reveal this breaking in production.

Done when: the edge case is handled correctly or fails safely, not just the normal case.

Common mistakes

  • Residual connections are not simply a shortcut that skips computation entirely; the layer still learns and contributes a meaningful transformation, the residual connection just makes it easier for gradients to flow and for the network to default toward the identity when that's the better choice.
