Neural Networks · Chapter 8

Architectures

Neural network architectures are specific, named patterns of how layers are structured and connected, such as convolutional networks for spatial data or transformers for sequential data, each designed with structural assumptions that fit particular kinds of problems.

Calculating reading timeHands-on labQuiz

In plain terms

It is like choosing between different specialized tools, a scalpel, a saw, a wrench, each shaped for a specific job rather than one universal tool trying to do everything adequately.

How it works

In practice it comes down to each architecture family encoding a specific inductive bias, such as convolutional networks assuming nearby pixels are related and sharing weights across spatial positions, or transformers assuming any token might relate to any other token and using attention to let the model learn those relationships directly from data.

A concrete example

A convolutional network is well suited to image classification because it assumes local spatial patterns matter and reuses the same small filters across the whole image, while a transformer is well suited to language because it can directly model relationships between words that are far apart in a sentence.

Trade-offs

  • The core tension: An architecture with structural assumptions that match the data, like convolution for images, trains faster and needs less data than a more general-purpose architecture, but if the assumption doesn't fit the actual data, the same structural bias can hurt performance instead of helping

Try it in code

const architectureFit = {
  images: "convolutional (local spatial pattern assumption)",
  sequences: "transformer (any-token-to-any-token attention)"
};

Change the inputs and watch which assumption breaks first. The gap between what you expected and what you got is where the learning is.

Hands-on lab

Task: For two different real tasks, one involving images and one involving sequential text, explain which architecture family's structural assumption fits each and why.

  1. Build or configure it, using the code example above as your starting point.
  2. Test it against one normal case and one edge case, and write down both actual outputs.
  3. Note what you would log, and which single metric would reveal this breaking in production.

Done when: the edge case is handled correctly or fails safely, not just the normal case.

Common mistakes

  • There is no single best neural network architecture across all tasks; choosing an architecture is a decision about which structural assumptions actually match your data and problem, not about picking whichever architecture is currently most popular or state-of-the-art in a different domain.
