Neural Networks · Chapter 6

Normalization

Normalization in neural network training rescales the intermediate activations within a network, typically to have zero mean and unit variance per layer or per batch, which stabilizes and speeds up training of deep networks.

Calculating reading timeHands-on labQuiz

In plain terms

It is like recalibrating an instrument partway through a long measurement process, so small drifts in scale don't compound and throw off every reading that follows.

How it works

In practice it comes down to computing the mean and variance of activations across a batch or layer, rescaling those activations to a standard distribution, and applying learned scale and shift parameters so the network can still represent the original range if that turns out to be optimal.

A concrete example

Batch normalization applied after a layer in an image classifier keeps activation distributions stable as training progresses, which lets the network use a higher learning rate and converge faster than it would without normalization.

Trade-offs

  • The core tension: Normalization layers make deep networks measurably easier and faster to train, but they add computational overhead per layer and introduce a dependency on batch statistics that can behave differently between training and inference if not handled carefully

Try it in code

function batchNorm(activations, gamma, beta, eps = 1e-5) {
  const mean = mean(activations), variance = variance(activations);
  return activations.map(a => gamma * (a - mean) / Math.sqrt(variance + eps) + beta);
}

Change the inputs and watch which assumption breaks first. The gap between what you expected and what you got is where the learning is.

Common mistakes

  • Normalization layers are not just a minor implementation detail; removing them from a deep network architecture that was designed and tuned with them can significantly destabilize training, since the network's other hyperparameters were chosen assuming normalization's stabilizing effect.
