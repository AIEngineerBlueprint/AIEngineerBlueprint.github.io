Memory · Chapter 11

Evaluation

Evaluation of memory systems tests whether stored information is retrieved when relevant, ignored when irrelevant or expired, and updated correctly when it changes, not just whether the system sounds coherent.

Calculating reading timeHands-on labQuiz

In plain terms

It is like testing a filing system not just by whether it can find one obvious document, but by whether it correctly retires outdated ones.

How it works

In practice it comes down to test scenarios that check whether the system recalls the right fact when relevant, correctly ignores irrelevant or expired memory, and handles conflicting stored facts, measured with precision and recall style metrics on memory retrieval rather than only conversational fluency.

A concrete example

An eval set might include a case where a user's preference changed last month, and check whether the system uses the new preference instead of the outdated stored one.

Trade-offs

  • The core tension: Rich evaluation scenarios covering conflicting facts, expired data, and retrieval misses catch real failure modes but take longer to build than simple smoke tests that only check basic recall

Try it in code

const memoryEvalCases = [
  { name: "correct recall", query: "what editor do I use", expectFactId: "pref_editor" },
  { name: "respects expiry", query: "where do I work", expectFactId: null }, // expired
  { name: "uses updated fact", query: "preferred contact method",
    expectValue: "email" } // was "phone" before an update
];

Change the inputs and watch which assumption breaks first. The gap between what you expected and what you got is where the learning is.

Hands-on lab

Task: Write three memory eval cases: correct recall, correct forgetting of expired data, and correct handling of a fact that changed since it was stored.

  1. Build or configure it, using the code example above as your starting point.
  2. Test it against one normal case and one edge case, and write down both actual outputs.
  3. Note what you would log, and which single metric would reveal this breaking in production.

Done when: the edge case is handled correctly or fails safely, not just the normal case.

Common mistakes

  • A memory system passing a demo that remembers basic facts is not evidence it handles the harder cases of contradiction, expiry, and irrelevant-memory suppression, which is where most production failures occur.
