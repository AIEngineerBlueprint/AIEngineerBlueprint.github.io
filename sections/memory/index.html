Learning track

Memory

A production-focused path for mastering Memory through concepts, diagrams, hands-on labs, interview practice, and capstone work.

Track outcomes

By the end of this track, you should be able to reason from first principles, design the architecture, identify failure modes, build a working slice, and explain how the system behaves in production.

In plain words: AI memory is like a notebook for an assistant. Short-term notes help with the current conversation; long-term notes help future sessions remember stable preferences or facts. Memory improves continuity, but it also creates privacy, retention, and correctness risks.

Chapter 1 Memory types Memory types is the taxonomy engineers use to decide what an AI system should remember and for how long: transient task state, recent conversation, specific past episodes, and long-lived facts about a user or domain. Chapter 2 Conversation memory Conversation memory is the short-term record of the current session's back-and-forth, used so an assistant doesn't ask a user to repeat information they already gave a few turns earlier. Chapter 3 Semantic memory Semantic memory stores durable, reusable facts about a user, project, or domain, independent of any single conversation, so the system can recall a stable preference months later without replaying every past chat. Chapter 4 Episodic memory Episodic memory stores specific past events or sessions as distinct, retrievable records, so an agent can answer what was decided in a particular past interaction rather than only holding generalized facts. Chapter 5 Working memory Working memory is the temporary scratchpad an agent uses while actively performing one task, such as intermediate reasoning steps or a partially built plan, which is not meant to persist once the task ends. Chapter 6 Summarization Summarization compresses older memory content into a denser form so a system retains the gist of what happened without paying the full context cost of the original transcript. Chapter 7 Retrieval memory Retrieval memory treats stored memories like a search index: instead of loading everything a system has ever remembered, it retrieves only the entries relevant to the current request, similar to RAG but over an agent's own memory store. Chapter 8 Privacy Privacy in memory systems governs what personal or sensitive information gets stored, who or what can access it, how long it persists, and whether the user can see or delete it. Chapter 9 Retention Retention defines how long each category of memory is kept before it is automatically deleted or must be re-confirmed, preventing indefinite accumulation of stale or unnecessary data. Chapter 10 Forgetting Forgetting is the deliberate removal or decay of stored memory so it no longer influences future behavior, whether triggered by a retention policy, a user request, or detection that a fact is outdated. Chapter 11 Evaluation Evaluation of memory systems tests whether stored information is retrieved when relevant, ignored when irrelevant or expired, and updated correctly when it changes, not just whether the system sounds coherent. Chapter 12 Storage patterns Storage patterns are the concrete data structures used to persist memory: key-value pairs for simple facts, vector embeddings for semantic search, or relational and graph structures for facts with explicit relationships. Chapter 13 Operations Operations for memory systems means monitoring how memory behaves in production: growth rate, retrieval relevance, staleness, and whether deletion and correction requests actually take effect. Chapter 14 Memory lab The memory lab is a hands-on exercise to build and verify a minimal memory system: store, retrieve, update, and forget, each demonstrated with visible input and output rather than left as theory. Chapter 15 Mini project The mini project ships a small, complete memory layer for one real use case, with explicit storage, retrieval, retention, and forgetting rules documented well enough that another engineer could reuse it. Chapter 16 Capstone The capstone designs a complete memory architecture for a real product: which memory types are needed, how each is stored and retrieved, the retention and privacy rules, and how the system is evaluated and monitored in production. Chapter 17 Interview pack The memory interview pack prepares you to discuss memory system design credibly: the difference between memory types, how retrieval actually works, and how you would handle privacy, retention, and forgetting requirements under follow-up questions.
