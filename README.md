# AIEngineerBlueprint

AIEngineerBlueprint is a production-focused static HTML learning platform for AI engineers. It is designed for GitHub Pages at `https://AIEngineerBlueprint.github.io` and works without a backend.

## Vision

AIEngineerBlueprint helps engineering students, software engineers, platform engineers, full-stack engineers, DevOps/SREs, and engineering leaders learn AI engineering by building production-grade AI products. The goal is to move beyond passive tutorials and teach the judgment needed to design, implement, evaluate, deploy, secure, and operate real AI systems.

The platform connects first-principles learning with practical product work: LLM internals, RAG, agents, MCP, guardrails, cloud deployment, local AI, coding assistants, interview preparation, and complete product delivery paths.

## What is included

- 33 learning tracks and 500+ generated chapter pages.
- Static HTML, CSS, JavaScript, SVG assets, search index, sitemap, manifest, and service worker.
- Dark/light mode, instant command-palette search, LocalStorage progress, bookmarks, quizzes, reading time, breadcrumbs, previous/next navigation, keyboard shortcuts, and generated concept-flow maps.
- Production-oriented lessons covering LLM internals, RAG, MCP, agents, guardrails, AI security, AI governance strategy, responsible AI, AI in SDLC, AWS, Azure, local AI, coding assistants, and interview preparation.

## Local development

```bash
node scripts/generate-site.js
npm run serve
```

Open `http://localhost:8080`.

## GitHub Pages deployment

1. Create a repository named `AIEngineerBlueprint.github.io`.
2. Copy or push this project to the repository root.
3. In GitHub, enable Pages from the default branch root.
4. Visit `https://AIEngineerBlueprint.github.io`.

## Content generation

The site is generated by `scripts/generate-site.js`. Generated files are overwritten on every generation run, including `sections/**/*.html`, `index.html`, `glossary.html`, `bookmarks.html`, `search-index.json`, `sitemap.xml`, `manifest.webmanifest`, `service-worker.js`, and `.nojekyll`.

Do not edit generated HTML directly. Put durable changes in the source files:

- `scripts/generate-site.js` for layout, page structure, track/chapter lists, generator behavior, shared sections, topic diagrams, and generated assets.
- `scripts/concepts.js` for chapter-specific teaching content: definitions, analogies, fundamentals, examples, misconceptions, exercises, and optional hand-authored `quiz` questions.
- `scripts/hooks.js` for chapter hooks: the short production-failure story that opens a chapter, keyed by `sectionSlug/chapterSlug`.
- `assets/css/styles.css` and `assets/js/app.js` for presentation and browser behavior.

Every concept field renders exactly once per page. Fields without authored content are omitted rather than padded with generated filler — if a section is missing on a page, the fix is to author that field, not to add template text.

Then rerun:

```bash
npm run generate
```

You can run on a different port with:

```bash
PORT=3000 npm run serve
```

Generated pages are committed as plain static HTML so the platform can be hosted by GitHub Pages without a build step.

## Project status

The current generated platform includes 577 chapters across 33 tracks, ~569 chapter-specific concept entries, production-failure hooks for the priority tracks (RAG, LLM Fundamentals, AI Agents, Guardrails, Production AI Systems), running-example products for those tracks, topic diagrams for ~28 key lessons, setup-and-smoke-test guidance for cloud/local AI environments, and AWS/Azure deployment variants for major product labs.

Chapter pages follow a single narrative arc: failure hook → why it matters → plain-terms analogy → how it works → concrete example → trade-offs → code → lab → common mistakes → quiz → going deeper. Project chapters (labs, mini projects, capstones) use a build-brief layout instead, and the interview track has its own shape.

Useful next content pushes: hooks for the remaining tracks, hand-authored `quiz` entries for priority chapters, more topic diagrams, and richer fundamentals for seeded entries.
